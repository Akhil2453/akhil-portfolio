# Content Guide

## Project entry

```yaml
---
title: "Project title"
summary: "One-sentence portfolio summary."
category: "Robotics & Innovation"
period: "2026 – Present"
organisation: "Organisation name"
role: "Your role"
status: "active"
featured: true
image: "/images/uploads/project-cover.webp"
imageAlt: "Clear description of the image"
tools: ["Tool 1", "Tool 2"]
outcomes:
  - "Outcome one"
  - "Outcome two"
links:
  - label: "Public reference"
    url: "https://example.com"
order: 99
---
```

Recommended body structure:

```markdown
## Context
## Problem
## Role and responsibilities
## Approach
## Technical or academic decisions
## Outcomes
## Reflection and next steps
```

## Blog entry

```yaml
---
title: "Article title"
description: "Search and social description."
pubDate: 2026-07-12
updatedDate: 2026-07-13
category: "Data Visualisation"
tags: ["Power BI", "Data Storytelling"]
image: "/images/uploads/article-cover.webp"
imageAlt: "Description of the cover"
draft: true
featured: false
downloadableResource: "/resources/example-resource.pdf"
---
```

Set `draft: false` only after final review.

## Update entry

```yaml
---
title: "Update title"
summary: "Short explanation of the update."
date: 2026-07-12
platform: "LinkedIn"
url: "https://www.linkedin.com/posts/..."
image: "/images/uploads/update.webp"
imageAlt: "Description"
featured: true
---
```

## Writing standard

- Use UK English.
- Explain the problem and contribution before listing tools.
- Use precise evidence instead of generic claims.
- Distinguish personal work from team work.
- Do not publish confidential client data or internal university documents.
- Do not publish student photographs without consent.
- Replace institutional data with simulated data for public dashboards.
