A dashboard is not a collection of charts. It is an interface for deciding what deserves attention. The difference is visible in the order of information.

## Layer 1: What is happening?

The first screen should establish the current state through a small number of key measures. In a sales dashboard, this may include revenue, gross profit, margin, target variance and growth. In an academic dashboard, it may include learner progress, course readiness, assessment completion and risk indicators.

The objective is orientation, not detail.

## Layer 2: Where is the change coming from?

The second layer breaks the headline metrics into useful dimensions. Region, product, department, course, cohort, owner or time period may explain the change. Visuals should be chosen according to the decision: comparison, trend, composition, distribution or relationship.

## Layer 3: What requires action?

The dashboard should help the viewer identify exceptions. These may include underperforming products, missed academic milestones, unusual costs, declining conversion rates or delayed deliverables.

Conditional formatting is useful only when it communicates a rule. Excessive colour creates noise.

## Layer 4: What evidence supports the decision?

Users need access to the underlying records. A drill-through page or detailed table can show the transactions, tasks, learners or items behind the metric. This makes the dashboard auditable and reduces arguments about the source of the number.

## Design for a conversation

A strong dashboard supports a sequence of questions:

1. What changed?
2. Where did it change?
3. Why might it have changed?
4. Which records require attention?
5. What action should be taken?

When the visual structure follows this sequence, the dashboard becomes part of an operating process rather than a decorative report.
